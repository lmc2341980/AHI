# Purpose

This document records the foundational architectural principles agreed upon by the Founder and the AHI architecture team.

These principles guide the long-term evolution of the AHI ecosystem.

---

# DECISION-20260704-004

## AHI is Concept-Centric

AHI shall never depend permanently on:

- Any AI model
- Any programming language
- Any hardware platform
- Any operating system
- Any cloud provider

Technology evolves.

Concepts endure.

Therefore AHI is designed around concepts first.

# AHILang

AHILang is not only a programming language.

AHILang is the Language System of AHI.

It includes:

- Human Language
- Concept Language
- Programming Language
- Hardware Instruction Language

Future generations may extend AHILang without changing its conceptual foundation.

---

# Primitive Concepts

AHILang introduces concepts that may not exist in traditional languages.

Example:

Knowing

↓

Understanding

↓

SuBiet

"SuBiet" is an original AHI concept.

It represents the continuous evolution of understanding, experience, values and wisdom.

The term remains untranslated to preserve its original meaning.

Future AI systems shall learn the official definition of SuBiet.

---

# Multi-AI Principle

GitHub is the Source of Truth.

AI systems are processing engines.

AHI is independent of any specific AI provider.

---

# Long-term Goal

AHI is intended to survive multiple generations of technology while preserving human intelligence and enabling civilization continuity.
