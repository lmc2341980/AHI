# Purpose

This document defines the standard Workspace Profile (WSP) for the Artificial Hybrid Intelligence (AHI) ecosystem.

The Workspace Profile is the canonical identity card of an AHI Workspace. It provides the minimum approved information required for any Human, Artificial Hybrid Intelligence, Robot or Agent to reconstruct the current project context before beginning work.

The Workspace Profile complements the Workspace Boot Sequence by defining **what must be loaded**, while the Boot Sequence defines **how it is loaded**.

---

# Core Principle

Every Workspace shall be self-describing.

A participant shall understand the Workspace without relying on the memory of another participant or a previous conversation.

---

# Workspace Identity

Every Workspace shall define:

* Workspace Identifier
* Workspace Name
* Description
* Repository Owner
* Current Version
* Current Status
* Creation Date
* Last Updated
* Workspace Language
* Default Time Zone

---

# Workspace Purpose

The Workspace shall describe:

* Mission
* Vision
* Strategic Objectives
* Scope
* Expected Outcomes

---

# Repository Inventory

The Workspace shall maintain an approved inventory of all repositories.

For each repository record:

* Repository Name
* Repository Description
* Repository Purpose
* Repository Type
* Repository URL
* Default Branch
* Current Version
* Status
* Dependencies
* Maintainer

This inventory becomes the primary navigation map of the ecosystem.

---

# Repository Dependency Graph

The Workspace shall describe:

* Parent repositories
* Child repositories
* Shared repositories
* External dependencies
* Required repositories
* Optional repositories

The dependency graph enables automated project reconstruction.

---

# Workspace Roles

The Workspace identifies the active collaboration roles.

Typical roles include:

* Founder
* Project Manager (Quản lý dự án)
* Professor (Chuyên gia)
* Team Leader (Trưởng nhóm)
* Secretary (Thư ký)
* Assistant (Trợ lý)

Each role shall define:

* Responsibilities
* Permissions
* Review authority
* Decision authority

---

# Workspace Language Profile

The Workspace shall define:

* Default language
* Supported languages
* Translation rules
* Terminology standards
* Abbreviation policy

Abbreviations shall always include explanations appropriate to the language and audience.

Artificial Hybrid Intelligence shall communicate according to the user's:

* native language;
* cultural context;
* age;
* education;
* profession;
* communication preferences.

---

# Active Project Status

The Workspace shall identify:

* Current milestone
* Current sprint
* Active repositories
* Active documents
* Open discussions
* Pending reviews
* Pending decisions
* Blockers
* Risks

---

# Memory Configuration

The Workspace shall declare the available memory layers:

* Working Memory
* Session Memory
* Draft Memory
* Discussion Memory
* Decision Memory
* Knowledge Memory
* Evolution Memory

Each layer shall define its retention policy, access permissions and synchronization rules.

---

# Knowledge Sources

The Workspace shall identify the approved knowledge sources.

Examples include:

* GitHub repositories
* Approved standards
* Approved specifications
* Intelligence Assets
* Repository Boot Manifest
* Workspace Manifest
* Architecture documents

Unapproved information shall never replace approved knowledge.

---

# Governance Profile

The Workspace shall define:

* Governance model
* Approval workflow
* Versioning rules
* Review process
* Change management
* Audit requirements

---

# Collaboration Configuration

The Workspace shall support collaboration among:

* Individuals
* Families
* Organizations
* Governments
* Educational institutions
* Research institutes
* Artificial Hybrid Intelligence systems
* Robots

All collaboration shall follow the same constitutional principles.

---

# Evolution Profile

The Workspace continuously records:

* New repositories
* New standards
* New Intelligence Assets
* Architectural evolution
* Governance evolution
* Lessons learned
* Historical milestones

Evolution shall always preserve previous approved states.

# Long-term Vision

Every AHI Workspace shall become a living digital environment capable of reconstructing itself from approved knowledge.

Future participants shall not inherit isolated documents.

They shall inherit a complete, understandable and continuously evolving Workspace that preserves the history, structure, governance and intelligence of the ecosystem across generations.
