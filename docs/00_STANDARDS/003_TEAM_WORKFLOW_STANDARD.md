# Purpose

This document defines the standard collaboration workflow between humans, AI assistants and future AHI systems.

Every participant in the AHI ecosystem shall follow the same workflow to ensure consistency, traceability and continuous knowledge evolution.

GitHub is the Source of Truth.

---

# Team Roles

## PM — Project Manager

Responsible for:

- Defining project objectives.
- Setting priorities.
- Making final project decisions.
- Approving implementation plans.
- Coordinating repositories and teams.

The PM focuses on project direction and execution.

---

## P — Professor (Expert)

Responsible for:

- Research.
- Architecture.
- Technical analysis.
- Innovation.
- Professional recommendations.
- Identifying risks and opportunities.

The Professor provides expert knowledge but does not make final decisions.

---

## S — Secretary

Responsible for:

- Recording discussions.
- Standardizing documents.
- Maintaining GitHub documentation.
- Managing document metadata.
- Updating the Document Registry.
- Preserving project history.

The Secretary ensures that knowledge is never lost.

---

## L — Team Leader

Each Repository or major Folder shall have one Team Leader.

The Team Leader is responsible for:

- Reviewing technical quality.
- Maintaining repository standards.
- Coordinating specialists.
- Reviewing pull requests.
- Ensuring compliance with AHI standards.

Examples:

- L-Core
- L-Studio
- L-Governance
- L-Persons
- L-Sage
- L-Omniverse

---

## A — Assistant

The Assistant is the execution coordinator.

Responsibilities include:

- Collecting opinions from PM, P, L and S.
- Detecting conflicts.
- Summarizing proposals.
- Generating GitHub assets.
- Preparing implementation plans.
- Suggesting next actions.

The Assistant never makes final decisions.

The Assistant prepares decisions for approval.

---

# Standard Workflow

Every discussion follows the same workflow.

Question

↓

PM defines objectives

↓

P provides expert analysis

↓

L reviews repository-specific impacts

↓

S standardizes knowledge

↓

A integrates all opinions

↓

GitHub Asset generated

↓

Founder Approval

↓

DONE

↓

GitHub Commit

---

# Question–Answer Evolution Principle

Every question should contain a desired direction.

Every answer should generate the next meaningful question.

Knowledge evolves continuously through iterative questions and answers.

---

# GitHub Output Standard

Whenever implementation is required, the Assistant shall generate:

- Repository
- Folder
- File
- Document ID
- Commit Message
- Complete Markdown Content

No implementation task is considered complete without a GitHub Asset.
