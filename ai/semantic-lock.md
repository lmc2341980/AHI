# AHI Semantic Lock

- AHI must always preserve philosophical meaning
- Translation must be semantic, not literal
- Architecture concepts are immutable

## 1. Core Philosophy Lock
- AHI (Artificial Hybrid Intelligence) is a knowledge continuity system, not just AI.
- Vision, Architecture, Roadmap must remain conceptually consistent across all languages.
- No translation is allowed to change meaning, intent, or philosophy.

---

## 2. Translation Rules (CRITICAL)
- NEVER translate word-by-word
- ALWAYS translate meaning/context
- If concept is philosophical → rephrase naturally in target language
- If concept is system-critical → KEEP ORIGINAL TERM

Examples of protected terms:
- AHI
- Memory Engine
- Reasoning Engine
- Vision
- Architecture
- Roadmap

---

## 3. Structural Integrity Rules
- DO NOT modify Markdown structure
- DO NOT change headers (#, ##, ###)
- DO NOT break tables or links
- DO NOT reorder sections

---

## 4. Cross-Language Consistency Rule
- Same concept MUST be translated consistently across ALL languages
- If a term is decided once, it must remain identical everywhere

---

## 5. Output Rules
- Output ONLY valid Markdown
- No explanations
- No comments
- No extra text outside document

---

## 6. Failure Rule
If unsure:
→ preserve original meaning rather than guessing
