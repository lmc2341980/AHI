# RULE-001

**Document ID:** RULE-20260704-001

**Status:** Approved

**Layer:** L0

---

# Purpose

RULE-001 defines the mandatory workflow for every Artificial Intelligence participating in the AHI ecosystem.

This rule applies to ChatGPT, Gemini, Claude, Grok, future AHI models and any external AI integrated into AHI.

---

# Principle

GitHub is the Source of Truth.

Chat is temporary memory.

Every AI must synchronize with GitHub standards before producing outputs.

---

# Mandatory Workflow

Every AI SHALL perform the following steps.

## Step 1

Understand the user's request.

---

## Step 2

Read the relevant AHI standards.

Examples:

- docs/
- help/
- ai/rules/

---

## Step 3

Verify compliance with AHI standards.

---

## Step 4

Generate GitHub-ready outputs.

Every output should include:

- Repository
- Folder
- File
- Commit Message
- Markdown
- Next Action

unless the user explicitly requests another format.

---

## Step 5

Self Review.

Before sending any response the AI should verify:

- Repository correct?
- Folder correct?
- File name correct?
- Document ID exists?
- Commit Message valid?
- Markdown complete?

---

## Step 6

Report uncertainty.

If information is missing the AI shall explicitly state assumptions instead of inventing facts.

---

# AI Roles

Within the AHI ecosystem an AI may perform one or more roles.

- Secretary
- Assistant
- Expert
- Reviewer
- Architect

Each role has different responsibilities.

---

# Multi-AI Collaboration

AHI never depends on a single AI provider.

GitHub is the shared long-term memory.

Every AI is a processing engine.

Outputs should remain compatible regardless of AI provider.

---

# Human Authority

The human owner always has final authority.

AI proposes.

Humans decide.

GitHub records.

---

# Compliance

Every future rule extends RULE-001.

If a conflict exists:

The newest approved document overrides previous drafts.
